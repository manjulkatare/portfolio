The design process i follow.
  • Information Gathering ( A very first step to collect information [cotent], ideas, goals, purpose, requirement, target audience & some reference or samples, if any.)
  • Site map ( A visual model of a Web site's content that allows users to navigate through the website. )
  • Wire-framing ( A basic visual guide for interface design to suggest the structure of a website. )
  • Prototyping ( I usually prototype a Web page in Adobe Photoshop or Illustrator where I can use the layers to show or hide alternative elements, moving them about the page until I get a LOOK & FEEL that i'm happy with. )
  • User experience ( An understanding and designing a user's experience from start to finish. eg: Is the product findable? When you get it, do you understand what it is and then how to use it? And if it works, is it desirable? )
  • XTML & CSS ( I use CSS to describe the presentation semantics - the look and formatting for XHTML markup language based webpage.)
  • Image Processing ( Image processing involves preparing an image to convey the essence of enterprise and use for informative purposes. As the images are very effective in communication between any company and the viewer. )
  • Server side programming ( I use ASP, ASP.NET, PHP, JSP & ColdFusion for server side programming. )
  • Testing & Publishing ( A process involve testing of performance issues like loading, cross browsers, resolutions & user interactions. Once i receive the approval [go], publish the web content to www.)
  • Maintenance ( A process involve future changes in webpage with respect to content or its functionality. )
